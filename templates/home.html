Asynchronous Banking Transaction System Showcase

A Flask-based banking platform demonstration that features multithreaded transaction processing, SQLite persistence, and "worker-queue architecture".

Concurrent Processing

Worker-thread architecture processes all transactions asynchronously through a task queue system.

SQLite Persistence

Data is stored via Python's SQLite3 allowing for persistent logging to provide a complete account history and audit tracking.

Flask Service Architecture

Flask routes integrate with a dedicated backend service for clear seperation of work.
